# TableKit

A sort of mono-repository mobile app containing several utility apps for physical games I like to play

## Philosophy

The code put into this centralized mobile app should fall into the following principles;

1. We support English and Dutch equally inside the app, code and documentation are written in the en-gb.
2. A sub-app should never be an entire game, it should always expand upon existing foundations.
3. Where possible, refrain from using text and instead focus on the experience or visuals to display properties, rules and states.
4. If a game is available in the Netherlands, but the contents are not language dependent, then it may be introduced with it's dutch name in a translated form.
